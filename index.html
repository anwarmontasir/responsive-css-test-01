Responsive CSS Layouts

Welcome! My name is Anwar Montasir, and I’m a designer, a developer, and a teacher at Treehouse.

I’m glad you could join me in this exercise. Our initial goal will be making a responsive CSS layout using as little CSS as possible.

What is Responsive Web Design?

The term “Responsive Web Design” was coined by Ethan Marcotte in an article for A List Apart in 2010. Marcotte identified the need to use techniques like fluid grids, flexible images, and CSS media queries to keep content legible across a variety of viewports.

Although mobile devices and programming techniques have evolved considerably over the last ten years, the basic principles of Responsive Web Design are more or less unchanged from when Marcotte initially described them.

What will we be learning?

While our layout will grow more ambitious in future lessons, today we simply want a clean layout that remains legible as our viewport grows from mobile to tablet to desktop. A few techniques that will help us get there include the following:

CSS Reset

I prefer to start my CSS with a clean slate. I’ll be implementing a copy of Eric Meyer’s CSS reset at CDNJS.

System Font Stack

For my typeface, I’ll use a system default for faster loading and clean rendering. Learn more about system font stacks at CSS Tricks.

Fluid Typography

Want to increase type scale as your viewport width increases? Check out this article on fluid typography at CSS Tricks.
